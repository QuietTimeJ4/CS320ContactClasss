# CS320ContactClasss

How can I ensure I make the program people want and need? The program needs to be functional.
The program needs to do what the end user wants. Lastly, I need to know how to make it and do so.
How do I make all this happen?

The simple answer is testing. Going into more depth, the testing in CS-320 can help check
for functionality in the code, program, and software, and in the course I previously took,
I learned how to verify security. Both courses use different automated testing.

I interpret user needs using user stories, interviews, surveys, requirements, etc.
It doesn't really matter how I come to understand the needs of the user, so long as I
understand their needs. My personal preference is an interview with the one commissioning the work,
as they are the impetus, funding, and just as responsible for the basis of the design. If they don't
know what they want, I can help them find it, but I wouldn't make the idea alone. In the end, I am
the code/tester. The reason for the work needs to be from them.

When it comes time to make the program, I work with a goal in mind. I take the end goal, create major
goals, and sub-goals. I finish a goal, and then test the program I made before adding on top of it. I repeat
this process until I reach the end goal. After that, I test the finished product. If it passes, I am done, or
if it doesn't pass, I find the issue and fix it. The user can be involved, so I am not getting ahead of myself.

I don't want to overcomplicate an already complicated project by having too many protocols for creating 
code. So long as the coder makes legible and functional code, it doesn't matter how they do that. I have
to emphasize that the coder needs to make the code and not AI or copyrighted code. It would be my service
people would pay for.
